**PRD: 핵심인재 9-Box Matrix Builder**

## 1. 제품 개요

**제품명**  
핵심인재 9-Box Matrix Builder

**목적**  
핵심인재(COT) 또는 리더 후보군의 **성과 Performance**와 **잠재력 Potential** 데이터를 기반으로 9-Box Matrix를 자동 생성하고, 인재리뷰 회의에서 바로 활용 가능한 개인별 프로필, 조직별 분포, 육성 방향을 제공한다.

**사용자**  
- HRD 담당자  
- HRBP  
- 인재육성 담당자  
- 임원 인재리뷰 회의 준비자

**개발 방식**  
- Frontend: React  
- 데이터 관리: CSV 업로드 또는 JS mock data  
- Backend: 없음  
- 배포: 정적 웹앱, 사내 PC 또는 인트라넷 배포 가능

---

## 2. 문제 정의

현재 핵심인재 9-Box 작업은 보통 엑셀과 PPT로 수작업 처리된다.

주요 문제는 다음과 같다.

1. 평가 데이터, 잠재력 진단 데이터, 인재 프로필이 분산되어 있다.
2. 9-Box 분류 기준이 담당자마다 달라질 수 있다.
3. 인재리뷰 회의용 자료 제작에 시간이 많이 든다.
4. 조직별, 직급별, 직무별 인재 분포를 빠르게 비교하기 어렵다.
5. 특정 Box에 속한 인재의 육성 방향을 체계적으로 관리하기 어렵다.

## 5. 핵심 사용자 시나리오

### 시나리오 1. HRD 담당자가 9-Box를 생성한다

1. 사용자가 웹앱을 연다.
2. 샘플 CSV 양식을 확인한다.
3. 인재 데이터를 CSV로 업로드한다.
4. 시스템이 성과/잠재력 점수를 읽는다.
5. 사용자가 기준값을 확인하거나 조정한다.
6. 시스템이 자동으로 9-Box Matrix를 생성한다.
7. 사용자는 조직/직급 필터를 적용해 분포를 확인한다.

### 시나리오 2. 인재리뷰 회의에서 특정 인재를 확인한다

1. 사용자가 특정 Box를 클릭한다.
2. 해당 Box에 속한 인재 목록이 표시된다.
3. 사용자가 인재 카드를 클릭한다.
4. 상세 패널에서 평가, 역량, 경력, 강점, 리스크를 확인한다.
5. 사용자는 리뷰 코멘트를 메모한다.

### 시나리오 3. 회의자료로 출력한다

1. 사용자가 필터를 적용한다.
2. 9-Box와 통계 화면을 확인한다.
3. 출력 모드를 선택한다.
4. 브라우저 인쇄 또는 PDF 저장을 실행한다.

## 7. 9-Box 분류 로직

### 기본 점수 기준

성과 점수와 잠재력 점수는 1.0~5.0 척도를 기준으로 한다.

| 구분 | 기준 |
|---|---|
| Low | 1.0 이상 ~ 3.0 미만 |
| Medium | 3.0 이상 ~ 4.0 미만 |
| High | 4.0 이상 ~ 5.0 이하 |

### Box 좌표

- X축: Performance
- Y축: Potential

| Box ID | Potential | Performance | Box 명칭 |
|---|---|---|---|
| B1 | High | Low | Potential Talent |
| B2 | High | Medium | Growth Leader |
| B3 | High | High | Future Executive |
| B4 | Medium | Low | Inconsistent Performer |
| B5 | Medium | Medium | Core Contributor |
| B6 | Medium | High | High Performer |
| B7 | Low | Low | Under Fit |
| B8 | Low | Medium | Solid Professional |
| B9 | Low | High | Expert Performer |

### 분류 함수 예시

```js
function getLevel(score, thresholds) {
  if (score >= thresholds.high) return "High";
  if (score >= thresholds.medium) return "Medium";
  return "Low";
}

function getBox(performanceScore, potentialScore, thresholds) {
  const performance = getLevel(performanceScore, thresholds.performance);
  const potential = getLevel(potentialScore, thresholds.potential);

  const boxMap = {
    "High-Low": "B1",
    "High-Medium": "B2",
    "High-High": "B3",
    "Medium-Low": "B4",
    "Medium-Medium": "B5",
    "Medium-High": "B6",
    "Low-Low": "B7",
    "Low-Medium": "B8",
    "Low-High": "B9"
  };

  return boxMap[`${potential}-${performance}`];
}
```

## 8. 주요 화면

### 8.1 데이터 업로드 화면

**목적**  
CSV 파일을 업로드하거나 샘플 데이터를 불러온다.

**구성 요소**
- CSV 업로드 버튼
- 샘플 데이터 사용 버튼
- CSV 템플릿 다운로드 버튼
- 업로드 결과 요약
- 오류 컬럼 안내

**검증 조건**
- 필수 컬럼 누락 시 오류 표시
- 점수 값이 숫자가 아니면 오류 표시
- 성과/잠재력 점수가 범위 밖이면 오류 표시

---

### 8.2 9-Box Matrix 화면

**목적**  
인재를 9개 박스에 자동 배치한다.

**구성 요소**
- 3x3 Matrix Grid
- 각 Box 제목
- 각 Box 인원 수
- 인재 카드
- Box별 색상
- 성과/잠재력 축 라벨

**인재 카드 표시 정보**
- 이름
- 조직
- 직급
- 성과 점수
- 잠재력 점수
- COT 여부
- 승계후보 여부

---

### 8.3 필터/검색 패널

**목적**  
회의 목적에 맞게 대상자를 좁힌다.

**필터 항목**
- 조직
- 직급
- 직무군
- COT 여부
- 승계후보 여부
- Box
- 검색어

**동작**
- 필터 변경 시 Matrix, 통계, 목록이 즉시 갱신된다.
- 선택된 필터는 상단에 태그로 표시된다.
- 전체 초기화 버튼을 제공한다.

---

**목적**  
인재리뷰 중 개인의 주요 정보를 빠르게 확인한다.

**표시 항목**
- 기본정보
- 성과/잠재력 점수
- 리더십 점수
- 핵심 경험
- 강점
- 리스크
- 개발 필요점

**MVP 동작**
- 인재 카드 클릭 시 우측 패널 또는 모달로 표시
- 닫기 버튼 제공
- 출력 모드에서 상세 정보 포함 가능

---

### 8.5 요약 통계 화면

**목적**  
전체 인재 분포를 정량적으로 확인한다.

**표시 지표**
- 전체 인원 수
- COT 인원 수

- High Potential 비율
- High Performance 비율

## 19. 주요 리스크

| 리스크 | 대응 |
|---|---|
| 실제 데이터 컬럼명이 회사마다 다름 | CSV 템플릿 제공, 컬럼명 명확화 |
| 평가 기준이 조직마다 다름 | 기준값 설정 기능 제공 |
| 개인정보 보안 우려 | 서버 저장 없음, 로컬 브라우저 처리 강조 |

| 카드가 많아져 화면이 복잡함 | 필터, 검색, Box별 접기 기능으로 대응 |
